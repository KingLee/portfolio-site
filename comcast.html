DailyUI

Simple, cloud-enabled router settings.

A project while working with Comcast Cable

XFINITY Internet is a responsive web service for customers to access and control their home WiFi networks from any browser enabled device. There are ~3 million gateways that have this service enabled, and more are being added weekly.

Planning components first.

When thinking about a large scale service, there has to be a consolidation of design patterns that can be reusued in multiple instances. In this case, we thought about what similar types of data and actions we had throughout the service and developed modules around those. We came up with a few basic ones:

Setting a consistent interaction pattern.

We attempted to establish a consistent conceptual pattern for the user when they are changing a setting. The design was influenced by the limitation of the API calls to the gateway itself, so we decided to limit user interaction to a single "grouping" of settings through a dialog box, preventing a user from making changes to multiple "groups" of settings.

Applying the XFINITY visual style

Comcast is in the midst of overhauling it's visual language to match more closely to the X1 TV platform. This means a move towards darker blues, blurred background images, subtle gradients, and some refreshed global items, among other things.

Making it responsive.

With a giant company like Comcast, it is a rarity to find a group of people who consider their mobile users to be of utmost importance. We designed the service to work primarily on mobile first, and made transitions and design considerations for desktop second.

We were able to reuse a lot of the mobile editing patterns, almost 1-to-1, in the dialog boxes on desktop to keep both the interaction patterns consistent, and make life easier for our developers.
